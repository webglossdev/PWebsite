🚧 Em Desenvolvimento

O Quê e Como:

Uma ferramenta standalone de diagnóstico técnico criada com uma estética inspirada no Windows Vista Aero e "Skeuomorphism" moderno. Por Que:

Milestones:

  • Criar o layout standalone "Aero Glass" e responsivo.
  • Refatorar dados fixos para buscar dinamicamente com manipulação de DOM pelo JavaScript (Fetch API + JSON).
  • Implementar e polir as páginas complementares de "Avaliações" e "Trabalhos".
  • Separar totalmente em um subdomínio (Ex: ajuda.meusite.com).
  • Lançamento final para clientes.

Calendário Olímpico

🌐 Público
Carregando dados do GitHub...
HTML CSS JavaScript JSON (Base de dados) Vercel Tailwind CSS

O Quê e Como:

Um website que centraliza todas as olimpíadas científicas brasileiras do ano: datas de provas e inscrição, matérias que caem, séries que podem participar e links para estudo específico de cada prova. Toda a informação é alimentada por uma base de dados em JSON, tornando fácil atualizar e manter o conteúdo.

Por Que:

Eu participava de várias olimpíadas e notei que era muito difícil encontrar todas as informações organizadas em um só lugar — datas, matérias, séries permitidas, links de estudo. Decidi criar essa ferramenta para ajudar outros estudantes a não perderem nenhuma oportunidade.

Milestones:

  • Base de dados JSON com olimpíadas, datas, matérias e séries.
  • Interface com filtros dinâmicos (série, matéria, mês, modalidade).
  • Timeline vertical com cronologia de eventos.
  • Página de diretório com acordeões para cada olimpíada.
  • Deploy no Vercel com domínio próprio.

Servidor Local

🔒 Privado
Tailscale / ZeroTier One Minecraft Server (mcss) Playit.gg Jellyfin Docker Compose

O Quê e Como:

Serviços para mim e meus amigos rodando localmente na minha máquina: Jellyfin para streaming privado de filmes e séries (CD/BluRay Rips), Servidor de Minecraft utilizando o software "mcss" (MC Server Soft), Playit.gg usado para conseguir um IP para meus amigos acessarem o servidor sem que eu precise fazer redirecionamento de portas no roteador, Docker Desktop para virtualizar esses e outros serviços em containers seguros que posso fazer backups e gerenciar facilmente, e Tailscale/ZeroTier One para acessar remotamente minha rede local.

Por Que:

Serviços parecidos eram inexistentes, muito caros ou precários quando gratuitos, e tinha a questão da privacidade. Então decidi fazer eu mesmo; desse jeito tenho controle total sobre os serviços e os dados.

Dificuldades Enfrentadas:

Tentar fazer redirecionamento de portas antes de descobrir sobre o CGNAT (Carrier-Grade NAT) impedia o uso do meu IPv4 público. Como não tinha muito interesse em configurar IPv6 no momento (mas vou fazer no futuro para usar meu subdomínio diretamente), resolvi utilizando túneis com o Playit.gg e Tailscale/ZeroTier (estou fazendo a transição do ZeroTier para o Tailscale, já que o ZeroTier tem algum tipo de dificuldade com tráfego em tempo real TCP/UDP em algumas rotas).

Milestones:

  • Configurar e subir todos esses serviços na máquina atual.
  • Transicionar completamente do ZeroTier ao Tailscale e dominar as ACLs e funcionamentos internos do Tailscale.
  • Upgrade na minha máquina pra rodar tudo 24/7 (ela já fica ligada 24/7, mas a maior parte dos serviços não).
